## 正文

```
记一下我人生第一个 vibe coding 作品🍊

做了一个叫 DualRead 的 Chrome 插件,
专门给中文母语者学英语用。

——————————————————

为什么做这个👇

我平时看英文文章的时候,总是被一两个
生词卡住。开翻译软件→复制粘贴→看完
又回来继续读,节奏全断。

而且单词看完就忘了,生词本懒得记。

——————————————————

它能做什么👇

🖱️ 在任何英文网页上点一下单词,旁边
直接弹出中文翻译气泡。不用跳走。

📘 觉得这个词想记住?点气泡上的"保存"。
单词会存到你的生词本里。

✨ 之后你在别的网页上再碰到这个词,
它会被温柔地画上下划线,提醒你「哦
这个我查过」。复习完全被动发生。

📤 想做 Anki 闪卡?一键导出 CSV。

——————————————————

关于 vibe coding🤖

我本职不是程序员,这个东西从设计到
发布全是跟 Claude 一边聊一边写出来的。
写代码的时候更像在跟一个同事讨论,而
不是查文档查到头秃。

最神奇的感觉是:脑子里那个一直想做但
觉得"我肯定做不出来"的东西,它真的出
现在屏幕上了。

——————————————————

🔒 小声说一句:
不要账号、不要后端、不要广告。你选的
单词只会发给 Google 翻译,其他什么都
不离开你的浏览器。我自己都在用,我自
己肯定最在乎隐私🫡

——————————————————

目前刚提交到 Chrome 商店审核中,过审
后会在评论区补链接。有任何想要的功能
或者觉得哪里不好用,都欢迎来喷我👇

也想听听同样用 AI 自己做过小东西的
朋友,你们做了啥?

#vibecoding #AI编程 #Chrome插件
#学英语 #效率工具 #独立开发者
#第一次做产品 #学习工具分享
```

## 写作原则对照

| 原则                 | 在文案里的体现                                                            |
| -------------------- | ------------------------------------------------------------------------- |
| Outcome 先于 feature | 开头不说"这是一个 Chrome 插件",说"看英文被生词卡住"                       |
| 客户语言             | "节奏全断"、"看完就忘"、"懒得记" —— 用户自己抱怨的说法                    |
| 不过度承诺           | 没说"背单词神器"、"告别遗忘"、"效率翻倍";只说"温柔提醒"、"复习被动发生"   |
| 诚实 AI 角度         | 明说"我不是程序员"、"跟 Claude 聊着写出来" —— 2026 年这个角度比装大佬加分 |
| 次 CTA 埋互动钩子    | 结尾问「你们做了啥?」比「快来下载」转化低,但评论池打得开,算法会推         |

## ⚠️ 发布前必改/必核

1. **Chrome 商店链接** — 等审核通过,把「审核中」换成正式链接。审核中发出去,读者点不到就流失了。
2. **「3 周」写死在标题 B 和正文开头** — 如果实际做的时间不是 3 周,改成真实天数。小红书读者对具体数字敏感,评论区会被追问。
3. **「我本职不是程序员」** — 如果本职是程序员/相关专业,这句要改。假人设一次被识破翻不了身。
4. **评论区准备** — 发布后 2 小时内守在评论区,常见会被问的问题:
   - 为什么选 Google 翻译不用 DeepL?
   - 数据安全吗?会不会传到哪里?
   - 以后会加账号/收费吗?
   - 能支持其他浏览器吗?(Edge/Firefox)
   - vibe coding 用的什么工具?推荐零基础学吗?

如果更想走「独立开发者成长记录」路线而不是「工具种草」,文案会完全不同 —— 更多聊:

- 为什么选这个题目(个人痛点)

- 发布前夜的心情、第一个用户的反馈
- AI 辅助开发的边界(它能做什么、不能做什么)

这种风格适合建立长期账号人设,但首次发布转化率通常比种草文低一些。
